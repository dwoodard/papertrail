## 1. Model Relationships

**File:** `app/Models/Project.php`

### Added: User Relationship
```php
use Illuminate\Database\Eloquent\Relations\BelongsTo;

public function user(): BelongsTo
{
    return $this->belongsTo(User::class);
}
```

**Why:** Every project must have an owner. This enables:
- `$project->user` → get the owning user
- `$project->user->name` → access user data
- Query: `Project::with('user')->get()` → eager load owners

### Added: Places Relationship
```php
use Illuminate\Database\Eloquent\Relations\HasMany;

public function places(): HasMany
{
    return $this->hasMany(Place::class);
}
```

**Why:** A project contains many captured places (businesses). This enables:
- `$project->places()->count()` → how many places in this project
- `$project->places` → all places for this project
- Cascade delete: deleting project automatically deletes all its places

## 2. UUID Primary Keys (Custom Setup)

**File:** `app/Models/Project.php`

```php
use Illuminate\Database\Eloquent\Concerns\HasUuids;

class Project extends Model
{
    use HasFactory, HasUuids;

    protected $keyType = 'string';
    public $incrementing = false;
    
    // ... rest of model
}
```

**Why:** 
- `HasUuids` auto-generates UUID v4 on creation
- `$keyType = 'string'` tells Laravel the key is a string (not int)
- `$incrementing = false` disables auto-increment

**Benefits:**
- No sequential IDs exposed to users
- Works across databases (Postgres, SQLite, MySQL)
- Safe for distributed systems
- Better privacy (can't guess IDs)

## 3. Fillable & Casting Configuration

**File:** `app/Models/Project.php`

```php
protected $fillable = ['user_id', 'name', 'goal', 'status'];

protected $casts = [
    'created_at' => 'datetime',
    'updated_at' => 'datetime',
];
```

**Why:**
- `$fillable` whitelist prevents mass-assignment vulnerabilities
- `$casts` auto-converts to Carbon datetime objects
- Enables: `$project->created_at->diffForHumans()` → "2 hours ago"

## 4. Factory with Relationships

**File:** `database/factories/ProjectFactory.php`

```php
public function definition(): array
{
    return [
        'user_id' => User::factory(),
```

**Why:**
- `User::factory()` auto-creates a user if needed
- Enables testing without hardcoded IDs: `Project::factory()->create()`
- Can customize: `Project::factory()->for($user)->create()`
- Generates realistic fake data

## 5. Authorization Policy

**File:** `app/Policies/ProjectPolicy.php`

```php
use Illuminate\Foundation\Auth\Access\AuthorizesRequests;

class ProjectPolicy
{
    public function own(User $user, Project $project): bool
    {
        return $user->id === $project->user_id;
    }
}
```

**Usage in controllers:**
```php
$this->authorize('own', $project);
// Throws 403 Forbidden if not owner
```

**Why:** Ensures users can only access their own projects

## 6. User Model Enhancement

**File:** `app/Models/User.php`

```php
use Illuminate\Database\Eloquent\Relations\HasMany;

public function projects(): HasMany
{
    return $this->hasMany(Project::class);
}
```

**Enables:**
```php
$user->projects()->count()          // How many projects this user has
$user->projects()->where('status', 'active')->get()  // Active projects only
```

## Testing Customizations

**Test examples using customizations:**
```php
test('user can own projects', function () {
    $user = User::factory()->create();
    $project = Project::factory()->for($user)->create();
    
    expect($project->user_id)->toBe($user->id);
    expect($user->projects()->count())->toBe(1);
});

test('deleting user cascades to projects', function () {
    $user = User::factory()->create();
    $project = Project::factory()->for($user)->create();
    
    $user->delete();
    expect(Project::count())->toBe(0); // Cascaded delete
});
```

## What's NOT Customized (By Design)

- ❌ **Soft deletes** — Not needed; status field sufficient
- ❌ **Audit logging** — Timestamps capture create/update times
- ❌ **Versioning** — Projects are mutable, not versioned
- ❌ **JSON columns** — Not needed for this schema
- ❌ **Custom scopes** — Can add later if needed (e.g., `scopeActive()`)
